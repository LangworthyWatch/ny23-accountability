# LangworthyWatch Social-Card Brief Generator

Generate accessible design specifications for LangworthyWatch.org social-media
graphics. This skill produces the **specification** — not the graphic itself —
that prevents every downstream accessibility failure before design work begins.

> **⚠️ Ported and adapted from public-ledger 2026-06-08.** The upstream version
> targets the user's personal Beaudoin / Erie County Legislature D11 campaign
> (different palette, different community list, different brand). This LW
> version targets LangworthyWatch's fact-check social cards (1080×1080 PNGs
> in `social-media/`, generated by `create_*_card.py` Python/Pillow scripts).

## When to use

- User wants to create a social card for a published fact-check entry
- User mentions Facebook post, X/Bluesky card, Instagram, or web image for a fact-check
- User says `/campaign-graphic-brief` with a communication goal
- Before any `create_*_card.py` script work

## Arguments

`$ARGUMENTS` describes the communication goal. Examples:
- "SECURE Data Act / Big Brother contradiction"
- "nursing home staffing donations card"
- "rural hospitals — $212M offset Facebook card"
- "Energy Choice Act 4-corner pattern card"

## Context

LangworthyWatch fact-checks are presented side-by-side: what was said vs. what
was done. Social cards must convey the contradiction (or pattern, or missing
context) in a single glance, with primary-source citations visible.

## Step 1: Determine output type

| Goal | Format | Dimensions |
|------|--------|------------|
| Facebook square card | 1080×1080 | Most common |
| Instagram story | 1080×1920 | Stories / Reels |
| Open Graph image (web preview) | 1200×630 | `og_image_*.png` pattern |
| Twitter/X card | 1200×675 | 16:9-ish |
| Longer-form fact pattern card | 1200×1200 | Multi-track / pattern visuals |

For most fact-checks, default to a 1080×1080 Facebook card matching the existing
`social-media/create_*_card.py` pattern (see
`social-media/create_secure_data_act_card.py` as the most recent worked example).

## Step 2: Apply LW social-card palette with verified contrast

### Palette (from existing `create_*_card.py` scripts)

| Color | Hex | Use |
|-------|-----|-----|
| BG | `#F5F7FA` | Background (light gray) |
| NAVY | `#1E3A5F` | Header bar, topic headlines, donor-strip BG |
| DARK | `#1A202C` | Body text |
| WHITE | `#FFFFFF` | Text on dark / navy / red |
| RED | `#E53E3E` | CONTRADICTION verdict badge, "what he did" track headers |
| GREEN | `#276749` | "what he said" track headers, accurate / pro-position |
| ORANGE | `#C05621` | Warning callouts, "the pattern" emphasis |
| GOLD | `#D69E2E` | Donor-strip label, MISSING CONTEXT badge |
| MUTED | `#718096` | De-emphasized labels, footnote text |
| BORDER | `#E2E8F0` | Section divider lines, panel borders |

### Pre-verified contrast (approximate; verify before publication)

| Foreground | Background | Approx ratio | AA Normal (4.5:1) | AA Large (3:1) | Use for |
|---|---|---|---|---|---|
| DARK | BG | ~16:1 | ✅ PASS | ✅ PASS | Body text |
| NAVY | BG | ~12:1 | ✅ PASS | ✅ PASS | Topic headlines |
| WHITE | NAVY | ~13:1 | ✅ PASS | ✅ PASS | Header bar text |
| WHITE | RED `#E53E3E` | ~4.4:1 | ⚠️ borderline | ✅ PASS | Verdict badge — OK if text is bold/large |
| GOLD `#D69E2E` | NAVY | ~3.6:1 | ❌ FAIL | ✅ PASS large | Donor-strip label — only as large bold |
| RED | `#FFF5F5` (red track BG) | ~5:1 | ✅ PASS | ✅ PASS | Track labels |
| GREEN | `#EBF8F0` (green track BG) | ~5:1 | ✅ PASS | ✅ PASS | Track labels |
| ORANGE | `#FFFAF0` (orange callout BG) | ~4.5:1 | ✅ borderline | ✅ PASS | Pattern callout |

**Hard rule:** GOLD on NAVY (~3.6:1) is the riskiest pair — keep it as
bold-and-large only (Arial Bold 16pt+). For the donor-strip label use of
"DONOR FOOTPRINT — PACS WHOSE ORGS SIGNED THE H.R. 8413 LETTER", the existing
font size (~16pt bold) passes AA Large by a margin. Verify with WebAIM
calculator if scaling smaller.

## Step 4: County variants (LW analog of community variants)

When the brief calls for a county-specific series (e.g., a healthcare entry
that affects all 8 NY-23 counties with different facility names), generate
one spec per county.

### NY-23 county list (per `langworthy-tracker/CLAUDE.md`)

1. Allegany
2. Cattaraugus
3. Chautauqua
4. Chemung
5. Erie (partial — NY-23 includes the southern portion only)
6. Schuyler
7. Steuben
8. Tioga

### Variant template pattern

```
For each county in [requested list]:
  1. Resolve ${COUNTY} in headline / data line / call-out
  2. Generate county-specific alt text (/alt-text-writer)
  3. Verify no new contrast issues from name length changes
  4. Output separate script: create_[entry_slug]_[county].py
  5. Output separate PNG: social-media/[entry_slug]_[county].png
```

Most LW social cards are district-wide (single card per entry); county variants
are reserved for entries where the data meaningfully differs by county
(e.g., RHTP hospital allocations, IDA project totals).

## Step 5: Draft alt text

Every specification includes alt text drafts (see `/alt-text-writer` for the
full skill).

**Short alt (≤125 characters)** — for Facebook/social:
```
LangworthyWatch fact-check social card: [Topic] — [Verdict]. [One-line summary]. Includes [data callout].
```

**Long description (2-3 sentences)** — for web:
```
Social card from LangworthyWatch.org documenting [the fact-check topic].
[Verdict] verdict. [Key data points]. Cites [primary sources]. Full fact-check
at langworthywatch.org/fact-checks/[slug]/.
```

Readability target: Grade 8 or below (Flesch-Kincaid).

```markdown
## Accessibility Checklist
- [ ] All text pairings pass WCAG AA (4.5:1 normal / 3:1 large)
- [ ] Verdict badge text is large/bold (AA Large fallback for the ~4.4:1 RED bg)
- [ ] GOLD on NAVY only used at large bold sizes
- [ ] Alt text present
- [ ] Alt text ≤125 chars (social) or full description (web)
- [ ] County names verified against NY-23 list (if county-variant series)
- [ ] LangworthyWatch.org footer URL points to the actual published fact-check

## Generation
- Script: `social-media/create_[slug]_card.py`
- Pattern reference: `social-media/create_secure_data_act_card.py` (most recent)
- Output: `social-media/[slug].png`
- Re-render command: `python3 social-media/create_[slug]_card.py`
```

## Accessibility gates — BLOCKS output if:

1. Any text-on-background pair fails 4.5:1 (normal text) AND the text is below
   18pt regular / 14pt bold (i.e., not "large text" per WCAG)
2. Verdict badge text is below 18pt bold (not enough margin on the ~4.4:1 red bg)
3. GOLD #D69E2E used on NAVY at < 16pt or non-bold
4. Missing alt text for any variant
5. County name (in variant series) not in the NY-23 list
6. Footer URL doesn't match the published fact-check path

## Do NOT

- Produce actual graphics (this skill generates specifications only;
  `create_*_card.py` produces the PNG)
- Use color combinations not in the LW palette table without computing contrast
- Skip alt text for "internal" or "draft" graphics
- Mix the Beaudoin / ECL D11 campaign palette (#1B2A4A / #F5F0E8 / #C41E3A)
  into LW cards — different visual brand
- Assume font sizes are adequate without checking the Step 3 table
- Generate county variants for counties outside NY-23
